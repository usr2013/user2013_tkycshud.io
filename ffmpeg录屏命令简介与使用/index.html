    Ffmpeg录屏命令简介与使用

    以下是使用 ffmpeg 进行屏幕录制的详细指南，涵盖 Windows、macOS 和 Linux 系统，并提供常见场景的解决方案：

    一、通用录制参数解释

    在命令中常用的核心参数： - -f：指定输入/输出格式（如 gdigrab、x11grab、avfoundation）。 - -video_size：屏幕或区域的分辨率（如 1920x1080）。 - -framerate：帧率（建议 25~60，根据需求调整）。 - -i：输入源（如屏幕设备编号或音频设备）。 - -c:v 和 -c:a：指定视频/音频编码器（如 libx264、aac）。 - -crf：画质控制（18~28，值越小画质越高，文件越大）。

    二、各系统详细命令

    1. Windows 系统

    (1) 录制全屏（无音频）

    ffmpeg -f gdigrab -framerate 30 -i desktop output.mp4

    (2) 录制指定窗口（根据窗口标题）

    ffmpeg -f gdigrab -framerate 30 -i title="Chrome" output.mp4

    (3) 录制屏幕 + 系统音频（需安装虚拟音频设备，如 VB-Cable）

    ffmpeg -f gdigrab -framerate 30 -i desktop -f dshow -i audio="虚拟音频设备名" -c:v libx264 -c:a aac output.mp4
    • 查找音频设备名：ffmpeg -list_devices true -f dshow -i dummy

    (4) 硬件加速（NVIDIA GPU）

    ffmpeg -f gdigrab -framerate 30 -i desktop -c:v h264_nvenc -preset fast output.mp4

    2. macOS 系统

    (1) 录制全屏（默认设备号 1）

    ffmpeg -f avfoundation -i "1" -r 30 output.mp4

    (2) 录制屏幕 + 麦克风音频

    ffmpeg -f avfoundation -i "1:0" -r 30 output.mp4

    (3) 指定屏幕区域（通过 crop 滤镜）

    ffmpeg -f avfoundation -i "1" -vf "crop=800:600:100:200" -r 30 output.mp4
    • crop=宽度:高度:起始X:起始Y

    (4) 硬件加速（Apple VideoToolbox）

    ffmpeg -f avfoundation -i "1" -c:v h264_videotoolbox -b:v 5000k output.mp4

    3. Linux 系统（X11/Wayland）

    (1) X11 环境（全屏录制）

    ffmpeg -f x11grab -video_size 1920x1080 -framerate 30 -i :0.0 output.mp4

    (2) 录制屏幕 + 系统音频（PulseAudio）

    ffmpeg -f x11grab -video_size 1920x1080 -framerate 30 -i :0.0 -f pulse -i default -c:a aac output.mp4

    (3) 指定录制区域

    ffmpeg -f x11grab -video_size 800x600 -framerate 30 -i :0.0+100,200 output.mp4

    (4) Wayland 环境（需 wf-recorder 或 xdg-desktop-portal）

    wf-recorder -f output.mp4  # 第三方工具辅助

    (5) 硬件加速（VAAPI）

    ffmpeg -vaapi_device /dev/dri/renderD128 -f x11grab -video_size 1920x1080 -i :0.0 -vf 'hwupload,scale_vaapi=format=nv12' -c:v h264_vaapi output.mp4

    三、高级场景

    1. 无损录制（超大文件，适合后期编辑）

    ffmpeg -f x11grab -video_size 1920x1080 -framerate 30 -i :0.0 -c:v ffvhuff lossless.avi

    2. 录制鼠标光标（Windows）

    ffmpeg -f gdigrab -draw_mouse 1 -i desktop output.mp4

    3. 多显示器录制（Linux）

    ffmpeg -f x11grab -video_size 3840x1080 -i :0.0+0,0 -vf "crop=1920:1080:0:0" output.mp4

    4. 直播推流（同时录制并推流）

    ffmpeg -f x11grab -i :0.0 -f pulse -i default -c:v libx264 -preset fast -c:a aac -f flv "rtmp://live.twitch.tv/app/your-stream-key"

    四、常见问题解决

    1. 录制卡顿

    • 降低帧率（-framerate 25）
    • 使用更快的编码预设（-preset ultrafast）
    • 启用硬件加速（如 h264_nvenc）

    2. 无声音

    • 确认音频设备名称是否正确。
    • macOS 可能需要权限：系统设置 → 隐私与安全性 → 屏幕录制 → 允许终端/FFmpeg。

    3. 文件过大

    • 提高 -crf 值（如 -crf 28）。
    • 限制码率（-b:v 2000k）。

    五、结束录制

    • 正常停止：按 q 键或 Ctrl+C。
    • 强制停止：Ctrl+Z（可能导致文件损坏）。

    通过调整参数和编码器，可以平衡画质、性能和文件大小。建议先测试短时间录制，再逐步优化参数。
